# 学習ドリルアプリ量産スキル (App Duplication Skill)

このファイルは、本プロジェクト（`shousuukakezan`）の完成されたゲームシステムやUIデザインを引き継ぎ、**別の単元のアプリを最速で作成・公開するためのAI向け指示書（スキル）**です。
次回以降、AIにアプリを作らせる際にこのファイルの内容を渡すか、「Skillsフォルダのスキルを使って作って」と指示してください。

---

## 🤖 AIへの指示用テンプレート（そのままコピペしてAIに送信してください）

```text
現在のプロジェクトの構造（HTML/CSS/JS）をベースにして、新しい学習ドリルアプリを作成してください。
以下の要件に従い、必要なファイルを修正し、GitHubの新しいリポジトリへPushしてください。

【今回のアプリ要件】
・新しいリポジトリ名（フォルダ名）: [例: bunsuu-tashizan]
・アプリのテーマ名: [例: 分数の足し算]
・GASの記録先シート名（ランキング動的生成用）: [例: bunsuu]
・問題生成のルール: [例: 分母が同じ分数の足し算。答えは必ず約分すること等]

【AIが守るべき実装スキル（必須）】
1. ベースコードの維持:
   - Roblox風のポップな3Dブロック調UI
   - 3ライフ制のサバイバルモード（5問連続正解で全回復）
   - 「記録してやめる」機能とHUDのレイアウト
   - これらは完成されているため、構造やCSSは変更しないでください。
2. バックエンド（スプレッドシート・ランキング）の完全継承:
   - スコア保存やランキング表示には、既存のGoogle Apps Script (GAS) 経由でGoogleスプレッドシートをバックエンドとして利用し続けること。
   - `app.js` 内のGAS通信処理やランキング更新ロジックは破壊せず、そのまま引き継ぐこと。
3. 変更するファイル:
   - `problem-generator.js`: `topicName`の変更と、`generate()`内のロジックを【問題生成のルール】に従って完全に書き換える。
   - `app.js`: 冒頭の `APP_SHEET_NAME` を指定されたシート名に変更する。
4. 自動引き継ぎの確認:
   - ブラウザのタブ名（`<title>`）は `app.js` 内で自動的に `topicName` に書き換わる仕組み（`document.title = window.ProblemGenerator.topicName;`）になっているため、HTML側の変更は不要です。
```

## 📝 スキルの概要（人間の確認用）
- **UI/UX**: `index.css` に集約。太い枠線とドロップシャドウを使ったゲーム風デザイン。レスポンシブ対応済み。
- **バックエンド**: GASとGoogleスプレッドシートを利用したランキングシステム。アプリ間の成績をシートごとに一元管理する仕組み。
- **ゲームロジック**: `app.js` に集約。手書きキャンバスの制御、GASとの非同期通信、ライフやスコアの管理。
- **問題ロジック**: `problem-generator.js` に集約。アプリの「中身（問題と答え）」だけをここで独立して管理。このファイルだけ書き換えればどんなドリルにも変身する設計。
